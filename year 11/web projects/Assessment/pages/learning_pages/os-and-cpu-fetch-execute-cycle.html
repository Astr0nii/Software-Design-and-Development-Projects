OS and the CPU fetch execute cycle

The operating system is an integral part of any computer system, therefore is must perform a certain group of functions and tasks to be efficient and not slow down a computer system. The CPU must also have a cycle to execute tasks, otherwise it would be misdirected and not function properly. Operating system
Some of the main functions of an operating system are: allowing hardware to talk to software, a consistent user interface for applications to use, controlling system resources, running system applications (sometimes multiple at once!). The operating system has alot of stuff to do, however thanks to modern CPUs all of this can be done really, really fast. Fetch execute cycle
For the CPU to function properly it needs a consistent cycle, thats where the fetch execute cycle comes into play. This cycle follows these steps: fetch the instruction from storage, decode into operation code and data addresses, copy operation code into the instruction register and addresses into the address register, address register sends data to the storage register, operation code is sent to the ALU (Arithemitic logic unit), execute the instructions, send the result to the accumalator, store results in primary storage. This cycle is the foundation of the computer system, so without it the computer wouldn't function.

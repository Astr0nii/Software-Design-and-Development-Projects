Software Development Approaches

Software development is a complex process, therefore these approaches have been developed to make life a bit easier for those who develop software on a daily basis. Each process has its own strengths and weaknesses depending on the task being attempted. These are the 5 main development approaches used in software development, however more do exist which aren't covered here. Structured
The Structured development approach consists of 5 main stages: Defining, Planing, Implementation, Testing and Maintaining. Defining begins with a system analyst (leader of the software team), discussing the current problems and the requirements for the new system.

Planing is a process where the needs of the user are indentified, the system analyst will examine the inputs, outputs, processes and proceadures of the present system. A report is then created using these findings to further assist the other steps.

Implementation starts with the specifications being given the the design team. This team will look at designing and programming the solution. Designers must break down the required processes into smaller modules, each module should be small and understandable. This modularity allows each module to be tested individually.

Testing
Maintaining relates to keeping the software up-to-date, fixing any bugs or issues that come up in the future. The user may also identify changes that are needed. Agile
The Agile approach begins with breaking tasks into small increments with minimal planning. This development approach uses interations which consist of the whole software development cycle in a period of one to four weeks. At the end of each interation the software is shown off with changes and suggestions being made. This approach may take multiple interations before the software is released. Prototyping
A prototyping approach is used when a company needs to gather information on how a system will be used and to get feedback before the real solution is made. Rapid Application Development (RAD)
A Rapid Application Development (RAD) solution uses drag and drop programming environments, existing code/routines and is designed for small teams and solutions. Enduser
The Rapid Application Development (RAD) approach consists of the user creating the software, rather than a developer doing it, this creates the benefit of minimal cost, no communication issues and a clear goal for the requirements of the software.
